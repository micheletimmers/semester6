Inleiding

Op het moment dat we het concept duidelijk hadden, konden we als groep eindelijk werken aan een prototype. We hebben de groep opgesplitst, zodat iedereen goed door kan werken. Samen met Luuk werk ik aan het kookgedeelte van de app. Dit doen we in React Native.

React Native is een programmeertaal waar ik nog geen ervaring mee heb. We maken gebruik van React, omdat dit gebruikt kan worden op Android en iOS apparaten. Dit zijn de twee bekendste besturingssystemen voor de telefoon (Tweakers, 2016), en Simac medewerkers gebruiken volgens Jan van der Sanden ook Android en iOS telefoons.

Voor de database wordt Firebase gebruikt, omdat dit heel makkelijk te koppelen is en mensen in de werkgroep hier ervaring mee hebben. Zelf heb ik informatie opgezocht over Firebase, zodat ik hier ook meer over te weten kom.

Aanpak

Eerst heb ik een tutorial gevolgd over React Native om de beginselen te leren. Na een aantal dagen oefenen ben ik begonnen aan de eerste klus voor de app. Daarna kreeg ik de taak om de login pagina te ontwerpen en te bouwen in React Native. Ik heb gewerkt met een user stories, zodat ik duidelijk voor ogen had waar ik aan moest werken en welke functionaliteiten in het design moeten worden geplaatst. Als Simac medewerker wil ik weten waar ik mijn inloggegevens achter moet laten, zodat ik kan inloggen en de app kan gebruiken Acceptatiecriteria:
- Tekstinvoer voor gebruikersnaam en wachtwoord
- Button om in te loggen
- Doorstroom naar de app wanneer er ingelogd is



Het ontwerp dat ik heb gemaakt, is gebaseerd op de guidelines van Apple. Deze guidelines heb ik doorgenomen, zodat ik ze kon inzetten voor het ontwerp.
Voor de login pagina heb ik gebruik gemaakt van de Simac huisstijl. De kleuren zijn van Simac, met een lichte gradient. De kleuren complimenteren elkaar, en de belangrijkste elementen springen eruit (Apple, zd.).
Met het ontwerp heb ik rekening gehouden met margin ruimte aan de linker en rechter kant. Zo is de login pagina duidelijk en makkelijk te lezen. De button is afgerond en full-width. Hierbij heb ik rekening gehouden met de guidelines. Deze geven aan dat de buttons het beste naar voren komen op de pagina, wanneer deze gelijk is aan de tekstinvoer regels (Apple, zd.).


De elementen die hierboven zijn benoemd, heb ik verwerkt in verschillende ‘Textinput’, en ‘View’ elementen. Dit was even goed uitzoeken hoe ik het moest weergeven, omdat sommige styling elementen niet werkten zoals bijvoorbeeld HTML/CSS. Hier heb ik veel geleerd over de mogelijkheden van React Native.
